2Convert for the dedicated server

The co-op host is stored under a placeholder GUID that dedicated servers don't recognize. To fix it, the host must join the server once (creating a fresh character) — the new file that appears in the server's Players/ folder is their real GUID. Enter it below.

The player file that doesn't match any co-op player is the host's new GUID.
What does the conversion do?

The 4-layer fix used by the community for post-1.0 saves: (0) the host's character file and world entry move to the new GUID; (1+2) every pal is re-keyed to the zero GUID (the server purges pals keyed to a non-existent player) and pal ownership fields are retargeted; (3) guild membership handles are corrected; (4) stale per-slot owner references in party/palbox containers are cleared. A verification pass re-parses the result before you can download it. Files are written in the zlib (PlZ) format, which the server accepts.
